# Phase 1a Implementation Plan — Dynamic Actor from DynamoDB

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the hardcoded Phase 0 WebFinger handler with DynamoDB-backed federation endpoints, a shared library, and a provisioning CLI.

**Architecture:** 8 Lambda handlers share an `ActivityPubCore` library for DynamoDB access and model types. The provisioning CLI seeds actor data. SAM template imports environment stack exports.

**Tech Stack:** Swift 6.3, aws-sdk-swift (AWSDynamoDB, AWSSSM), swift-crypto (_CryptoExtras), swift-argument-parser, AWS SAM

**Spec:** `docs/superpowers/specs/2026-03-28-phase-1a-design.md`
**Project plan:** `docs/PROJECT-PLAN.md` (Actor JSON-LD at lines 349-394, DynamoDB schema at lines 304-327)

```swift
import Foundation

public struct OrderedCollection: Codable, Sendable {
    public let context: String
    public let id: String
    public let type: String
    public let totalItems: Int
    public let first: String?
    public let last: String?
    public let orderedItems: [String]?

    enum CodingKeys: String, CodingKey {
        case context = "@context"
        case id, type, totalItems, first, last, orderedItems

            id: id, type: "OrderedCollection", totalItems: 0,
```

**Files:**
- Create: `Sources/ActivityPubCore/DynamoDBStore.swift`

- [ ] **Step 1: Create DynamoDBStore**

Thin wrapper around `DynamoDBClient`. Reads the `TABLE_NAME` environment variable. Uses `GetItem` to fetch actor profiles by PK/SK.

Key implementation details:
- Initialize `DynamoDBClient` with the region from `AWS_REGION` env var (set by Lambda runtime)
- `getActor(username:)` does `GetItem` with PK=`ACTOR#{username}`, SK=`PROFILE`
- Convert DynamoDB attribute map to `Actor` struct manually (DynamoDB returns `[String: DynamoDBClientTypes.AttributeValue]`)
- `actorExists(username:)` uses the same GetItem but only checks if the item exists (use `ProjectionExpression` to minimize data transfer)

### Task 5: ActorHandler

**Files:**
- Create: `Sources/ActorHandler/main.swift`

- [ ] **Step 1: Implement ActorHandler**

1. Extract `{username}` from `event.pathParameters["username"]`
2. Look up actor in DynamoDB
3. Return 404 if not found
4. Content-negotiate on `Accept` header:
   - `text/html` → 302 redirect to `https://{SERVER_DOMAIN}/@{username}`
   - Default → Actor JSON-LD
5. Build the full Actor JSON-LD matching PROJECT-PLAN.md lines 349-394 exactly:
   - Full `@context` array (AS2, security/v1, toot namespace inline object with discoverable, indexable, featured, featuredTags, attributionDomains, schema, PropertyValue, value, manuallyApprovesFollowers, sensitive)
   - `type: "Service"`
   - `id`, `preferredUsername`, `name`, `summary`, `inbox`, `outbox`, `followers`, `following`, `url`
   - `publicKey` block with `id: "{actorUrl}#main-key"`, `owner`, `publicKeyPem`
   - `discoverable`, `indexable: false`, `manuallyApprovesFollowers`
   - `published` (from `createdAt`)
   - `featured`, `featuredTags` URLs
   - `attributionDomains: [HANDLE_DOMAIN]`
   - `icon` block if `avatarUrl` is present

Note: Build the JSON manually (dictionary/string) rather than trying to model the full JSON-LD structure as a Codable type — the `@context` array with mixed types (strings and objects) is awkward in Swift's Codable system.

### Task 8: ActivityProvisioner CLI

**Files:**
- Create: `Sources/ActivityProvisioner/main.swift`

- [ ] **Step 1: Implement the provisioning CLI**

Use `ArgumentParser` for argument parsing. Required arguments: `--stage`, `--username`, `--display-name`. Optional: `--summary` (default empty), `--region` (default us-east-1).

Steps the CLI performs:
1. Generate RSA 2048 keypair using `_CryptoExtras.RSA` (import `_CryptoExtras` from swift-crypto)
2. PEM-encode the public key and private key
3. Create SSM SecureString parameter at `/activity/{stage}/keys/{username}` with the private key PEM as the value
4. Write actor profile to DynamoDB with:
   - PK: `ACTOR#{username}`, SK: `PROFILE`
   - All Actor model fields
   - `publicKeyPem`: the PEM-encoded public key
   - `privateKeyArn`: `/activity/{stage}/keys/{username}`
   - `createdAt`: current ISO 8601 timestamp
   - Numeric fields: 0
   - Boolean fields: `discoverable: true`, `manuallyApprovesFollowers: false`
5. Print success message with the actor's URI

Reference the `_CryptoExtras` API for RSA key generation — `_RSA.Signing.PrivateKey(keySize: .bits2048)`. Export public key via `.publicKey.pemRepresentation` and private key via `.pemRepresentation`.

```swift
import Testing
import Foundation
@testable import ActivityPubCore

@Test func webFingerResponseEncoding() throws {
    let response = WebFingerResponse(
        subject: "acct:test@happitec.com",
        links: [
            WebFingerLink(rel: "self", type: "application/activity+json", href: "https://activity.happitec.com/users/test"),
            WebFingerLink(rel: "http://webfinger.net/rel/profile-page", type: "text/html", href: "https://activity.happitec.com/@test"),
        ]
    )
    let encoder = JSONEncoder()
    encoder.outputFormatting = .sortedKeys
    let data = try encoder.encode(response)
    let json = try JSONSerialization.jsonObject(with: data) as! [String: Any]
    #expect(json["subject"] as? String == "acct:test@happitec.com")
    let links = json["links"] as! [[String: Any]]
    #expect(links.count == 2)
    #expect(links[0]["rel"] as? String == "self")
    #expect(links[0]["type"] as? String == "application/activity+json")
}

@Test func webFingerLinkOmitsNilFields() throws {
    let link = WebFingerLink(rel: "self", type: "application/activity+json", href: "https://example.com")
    let data = try JSONEncoder().encode(link)
    let json = try JSONSerialization.jsonObject(with: data) as! [String: Any]
    #expect(json["template"] == nil) // nil fields should not appear in JSON
}
```

Note: For the nil-omission test, the default `JSONEncoder` includes `null` for nil optionals. To omit them, either use a custom encoding strategy or accept `null` in the output.
