# Phase 1b Implementation Plan — CloudFront + Custom Domains

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add CloudFront distribution with full cache behaviors, custom domains via Route 53, and S3 media serving via OAC to the app stack. Add WebFinger redirect on happitec.com.

**Architecture:** CloudFront sits in front of API Gateway (execute-api origin with `/Prod` path) and S3 (via OAC). Custom cache policies for WebFinger and outbox query strings. Route 53 alias points the custom domain to CloudFront. Separate PR to happitec.com for the WebFinger redirect.

**Tech Stack:** AWS SAM, CloudFront, Route 53, S3 OAC, CloudFront Functions

### Task 4: WebFinger redirect on happitec.com (separate repo)

This task modifies the happitec.com repo at `/Users/spar/web-local/happitec.com/`.

**Files:**
- Modify: `/Users/spar/web-local/happitec.com/template.yml`

- [ ] **Step 1: Add WebFinger redirect CloudFront Function**

Add a new CloudFront Function resource to the happitec.com SAM template. The existing template already has `RewriteRequestStaticFunction` and `RewriteRequestSpaFunction` — this is a third function specifically for WebFinger.

```yaml
WebFingerRedirectFunction:
  Type: AWS::CloudFront::Function
  Properties:
    Name: !Sub "${AWS::StackName}-webfinger-redirect"
    AutoPublish: true
    FunctionCode: |
      function handler(event) {
        var request = event.request;
        if (request.uri === '/.well-known/webfinger') {
          var qs = request.querystring;
          var pairs = [];
          for (var key in qs) {
            pairs.push(encodeURIComponent(key) + '=' + encodeURIComponent(qs[key].value));
          }
          var qsString = pairs.join('&');
          return {
            statusCode: 302,
            statusDescription: 'Found',
            headers: {
              location: { value: 'https://activity.happitec.com/.well-known/webfinger?' + qsString }
            }
          };
        }
        return request;
      }
    FunctionConfig:
      Comment: "Redirect WebFinger requests to activity.happitec.com"
      Runtime: cloudfront-js-2.0
```

- [ ] **Step 2: Add a cache behavior for `/.well-known/webfinger`**

Add a new cache behavior to the existing `CloudFrontDistribution` resource, BEFORE the `DefaultCacheBehavior`. The behavior should match `/.well-known/webfinger` and associate the redirect function:

```yaml
CacheBehaviors:
  - PathPattern: /.well-known/webfinger
    TargetOriginId: bucketOrigin
    ViewerProtocolPolicy: redirect-to-https
    CachePolicyId: 4135ea2d-6df8-44a3-9df3-4b5a84be39ad  # CachingDisabled
    AllowedMethods: [GET, HEAD]
    FunctionAssociations:
      - EventType: viewer-request
        FunctionARN: !GetAtt WebFingerRedirectFunction.FunctionMetadata.FunctionARN
```

Note: The origin doesn't matter since the function returns a redirect before reaching the origin. Using `bucketOrigin` (the existing S3 origin) as a required placeholder.
